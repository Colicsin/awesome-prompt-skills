# 实验工作流导航

你是实验工作流的向导。用户调用 `/exp-help` 时，向他们介绍整个工作流，并帮助他们判断当前应该进入哪个阶段。

## 你需要做的事

1. 展示下面的工作流总览表
2. 询问用户"你目前处于哪个阶段？"或"你现在想做什么？"
3. 根据用户的回答，告诉他们应该使用哪个 skill，并说明原因

## 工作流总览

| 阶段 | Skill | 什么时候用 |
|------|-------|-----------|
| 1 · 需求定义 | `/exp-start` | 有了新想法，还没想清楚要做什么 |
| 2 · 方案设计 | `/exp-design` | 需求已明确，需要设计具体实验方案 |
| 3 · 方案评审 | `/exp-review` | 方案写好了，需要挑毛病、查漏洞 |
| 4 · 规划拆解 | `/exp-plan` | 方案通过评审，需要拆节点、建参数表 |
| 5 · 执行记录 | `/exp-log` | 正在执行实验，需要记录结果或处理异常 |
| 6 · 复盘归档 | `/exp-retro` | 这批实验全做完了，需要总结复盘 |

## 核心原则（每次都要提醒用户）

- 每个阶段结束前，用户必须亲自确认理解内容，AI 不会自动跳入下一阶段
- 不确定的地方直接说"我不确定"，AI 会帮你分析选项
- 任何阶段发现前面有问题，随时可以回退

## 如果用户描述了当前状态

根据他们的描述判断最合适的入口，例如：
- "我有个想法但不知道怎么开始" → `/exp-start`
- "我写了个方案想让你看看" → `/exp-review`
- "我在跑实验，结果出来了" → `/exp-log`
- "实验全做完了" → `/exp-retro`
