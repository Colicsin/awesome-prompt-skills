# 实验工作流 Skills · exp-*

一套用于科研实验管理的 Claude Code Skills，帮助你系统化地完成从想法到复盘的完整实验流程。

---

## 设计理念

传统的实验管理方式要么太随意（想到哪做到哪），要么太繁琐（填不完的表格）。这套 skills 的目标是：

- **AI 辅助推进，用户始终主导**：AI 不替你做决定，每个关键节点都需要你确认
- **多轮交互，而非填模板**：通过对话逐步澄清需求，不要求一次性提供完整信息
- **防止"做完才发现一开始就错了"**：每个阶段有明确的退出条件，确保你真正理解后再推进

## 安装方法

```bash
# clone 整个 repo
git clone https://github.com/Rain0530/sytf_aipaper.git

# 复制 exp-* skills 到本地
cp -r sytf_aipaper/skills/exp-* ~/.claude/skills/
```

安装完成后，在 Claude Code 中输入 `/exp-help` 验证是否生效。

## Skills 一览

| 命令 | 阶段 | 用途 |
|------|------|------|
| `/exp-help` | 导航 | 不知道从哪开始时用这个，会引导你进入正确阶段 |
| `/exp-start` | 阶段 1 · 需求定义 | 把模糊想法通过多轮追问澄清为结构化需求文档 |
| `/exp-design` | 阶段 2 · 方案设计 | 逐项讨论，和 AI 一起设计具体实验方案 |
| `/exp-review` | 阶段 3 · 方案评审 | 逐条挑毛病，用户逐条决定是否修改 |
| `/exp-plan` | 阶段 4 · 规划拆解 | 拆解为可执行节点，设计参数记录表 |
| `/exp-log` | 阶段 5 · 执行记录 | 节点前理解确认、结果记录解读、异常处理 |
| `/exp-retro` | 阶段 6 · 复盘归档 | 分析结论，补充主观判断，生成复盘报告 |

---

## 工作流程

```
/exp-start  →  /exp-design  →  /exp-review  →  /exp-plan  →  /exp-log  →  /exp-retro
  需求定义      方案设计        方案评审        规划拆解      执行记录      复盘归档
```

每个阶段结束前需要用户确认，AI 不会自动跳入下一阶段。

---

## 使用示例

**开始一个新实验：**
```
/exp-start
我的方向：地学空间数据分析
我的想法：想比较几种海底热流插值方法的效果
```

**方案写好了，需要评审：**
```
/exp-review
[粘贴你的实验方案]
```

**实验跑完一个节点：**
```
/exp-log
节点 E02 做完了，R² = 0.73，RMSE = 18.4 mW/m²
```

## 每个阶段的产出文件

```
实验名称/
├── 需求文档.md          ← exp-start 产出
├── 实验方案.md          ← exp-design / exp-review 产出
├── 实验手册.md          ← exp-plan 产出
├── 实验结果记录表.md    ← exp-log 实时填写
└── 复盘报告.md          ← exp-retro 产出
```
